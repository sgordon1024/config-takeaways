# Figma practice: the new product process

Two practice projects for the **Figma parts** of the process (the stages that actually touch Figma):

| Stage | Figma surface |
|---|---|
| 1. Framing | FigJam |
| 2. Context | Figma MCP + Code Connect, tokens |
| 3. Exploration | Figma Make, Motion, Shaders |
| 4. Build | Make / Make Local round-trip |
| 7. Final 5% | Motion mode, canvas |

- `complete/` is a fully worked reference for a sample app (**Tally**, an onboarding flow). Use it when you get stuck.
- `blank/` is the same workbook, empty. Fill it as we go.

These files are the brief, the prompts, and the capture sheet. The real work happens in your Figma account. Each file says what to do there and what to write down here.

## How to run it
Work `blank/` in order:
1. `brief.md` — the app and the bet (we set this in person).
2. `framing.md` — FigJam session.
3. `context.md` — skills, tokens, Code Connect map, golden set.
4. `exploration.md` — parallel directions in Make, then curate.
5. `build.md` — the Make to code round-trip.
6. `polish.md` — the final 5% on the canvas.

## The five Figma steps (quick map)
1. FigJam: cluster bets, capture the conviction brief.
2. Context: map your design system with Code Connect, wire the Figma MCP, write 2-3 skills.
3. Figma Make: generate 3 constrained directions, kill 2.
4. Make Local: build the survivor against your repo, publish to a branch. Pull back for visual tweaks via MCP + Code Connect.
5. Motion mode + canvas: hand-tune the feel and the empty-state voice.

Note: Figma MCP read-tools and Code Connect are seat-gated and only pay off once your system is mapped. A human owns every merge.
