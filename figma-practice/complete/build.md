# 4. Build: the Make to code round-trip

Two paths. Use the one that matches the tweak.

## Path A — stay in Make (structure and function)
1. Point **Make Local** at the Tally repo. The build is your repo, not an import.
2. Tweak by prompt, by direct controls, and by editing code layers.
3. Publish: push to a branch, or hand the change to a coding agent reading the system via MCP + Code Connect. A human merges.

## Path B — onto the canvas (visual and craft)
1. Rebuild the key screen as Figma components, or import the rendered build with a plugin. The code link is now broken.
2. Tweak visually plus Motion mode.
3. Back to code: there is no direct push. Map the components with Code Connect, expose them via MCP, have an agent regenerate the code. Lossy unless the system is mapped.

**Rule:** tweak the build in Make; tweak the system on the canvas.

## Practice goal
Make one functional change in Make and publish to a branch. Then make one visual change on the canvas and regenerate code from it. Notice which loop is tighter.
