# 1. Framing (drafted, ready for FigJam)

## Conviction brief (draft)
- **Problem:** new users sign up, never add a plant, and churn before any value lands.
- **Bet:** first plant added and watering schedule shown within 2 minutes.
- **Why now:** activation is the leak; the current welcome carousel teaches features before the user has a plant to care about.
- **Won't do:** no photo-ID onboarding, no account-completion nags, no community prompts before the first plant.
- **One metric:** Day-1 activation (added a plant, viewed a schedule they trust).

## Pressure-test (held, with one change)
The bet survived. Carry forward:
- Risk is schedule **trust**, not speed. → folded a trust signal into the definition of good.
- Assumes the user has a plant at signup. → watch for this in validation.
- Activation may not predict retention. → check week-1 adders vs week-4 retention before scaling.

## Candidate bets (drop these as FigJam stickies, then cluster)
- User adds their first plant and sees a schedule in under 2 min.  ← leading
- User sets up a room and adds plants to it.
- User turns on watering reminders.
- User identifies a plant from a photo.
- User browses the plant catalog.
